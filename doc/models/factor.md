
# Factor

*This model accepts additional fields of type object.*

## Structure

`Factor`

## Fields

| Name | Type | Tags | Description |
|  --- | --- | --- | --- |
| `Sid` | `string` | Optional | A 34 character string that uniquely identifies this Factor.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YF[0-9a-fA-F]{32}$` |
| `AccountSid` | `string` | Optional | The unique SID identifier of the Account.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^AC[0-9a-fA-F]{32}$` |
| `ServiceSid` | `string` | Optional | The unique SID identifier of the Service.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^VA[0-9a-fA-F]{32}$` |
| `EntitySid` | `string` | Optional | The unique SID identifier of the Entity.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YE[0-9a-fA-F]{32}$` |
| `Identity` | `string` | Optional | Customer unique identity for the Entity owner of the Factor. This identifier should be immutable, not PII, length between 8 and 64 characters, and generated by your external system, such as your user's UUID, GUID, or SID. It can only contain dash (-) separated alphanumeric characters. |
| `DateCreated` | `DateTime?` | Optional | The date that this Factor was created, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `DateUpdated` | `DateTime?` | Optional | The date that this Factor was updated, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `FriendlyName` | `string` | Optional | A human readable description of this resource, up to 64 characters. For a push factor, this can be the device's name. |
| `Status` | [`FactorEnumFactorStatuses?`](../../doc/models/factor-enum-factor-statuses.md) | Optional | The Status of this Factor. One of `unverified` or `verified`. |
| `FactorType` | [`FactorEnumFactorType?`](../../doc/models/factor-enum-factor-type.md) | Optional | The Type of this Factor. Currently `push` and `totp` are supported. |
| `Config` | `object` | Optional | An object that contains configurations specific to a `factor_type`. |
| `Metadata` | `object` | Optional | Custom metadata associated with the factor. This is added by the Device/SDK directly to allow for the inclusion of device information. It must be a stringified JSON with only strings values eg. `{"os": "Android"}`. Can be up to 1024 characters in length. |
| `Url` | `string` | Optional | The URL of this resource. |
| `AdditionalProperties` | `object this[string key]` | Optional | - |
